# Aethera Research Pipeline

A scraping and search engine that tracks **global tokenization updates** — solar energy, real world assets (RWA), carbon credits, DePIN, green bonds, and more. Built to give Aethera investors and admins real-time intelligence on what's happening in the tokenization world.

---

## What It Does

- Scrapes **RSS feeds** from CoinDesk, CoinTelegraph, PV Magazine, CleanTechnica, Ledger Insights, and more
- Scrapes **Google News RSS** for the latest tokenization stories worldwide
- Optionally hits **NewsAPI** and **Google Custom Search** for broader coverage
- Auto-classifies articles by **category** (solar, rwa, carbon, energy, depin, regulation...)
- Auto-detects **region** (North America, Europe, India, Middle East, etc.)
- Scores each article by **relevance** using keyword weights
- Stores everything in a local **SQLite database** with full-text search
- Exports to **JSON** for easy browsing in VS Code

## Setup

```bash
cd research-pipeline
npm install
cp .env.example .env
```

Optional — add API keys to `.env` for broader coverage:
```
NEWS_API_KEY=your_newsapi_key        # https://newsapi.org (free tier available)
GOOGLE_API_KEY=your_google_key       # Google Custom Search API
GOOGLE_SEARCH_ENGINE_ID=your_cx_id
```

## Commands

All commands run from inside `research-pipeline/`.

### Scrape

```bash
# Scrape default topic (solar energy tokenization)
npm run scrape

# Scrape a custom query
npm run scrape -- -q "carbon credit blockchain India"

# Scrape ALL 10 tracked topics at once
npm run scrape -- --all
```

### Search

```bash
# Full-text search across scraped articles
npm run search -- "solar tokenization"

# Filter by category
npm run search -- --category rwa
npm run search -- --category solar
npm run search -- --category regulation

# Filter by region
npm run search -- --region Europe
npm run search -- --region "North America"

# Combine filters
npm run search -- "carbon credits" --category carbon --region India

# Show more results
npm run search -- "tokenization" --limit 50

# Articles since a date
npm run search -- --since 2026-01-01
```

### Export to JSON

```bash
# Export everything to data/articles.json (open in VS Code)
npm run export

# Export to a custom path
npm run export -- -o ./data/my-snapshot.json
```

### Report

```bash
# Generate a trend report (category/region/source breakdown + top articles)
npm run report
```

### List Tracked Topics

```bash
npm run scrape -- --help   # shows options
npx tsx src/cli.ts topics  # lists all 10 tracked topics
```

### Start the API Server

```bash
# Runs REST API on port 4100
npm run serve
```

### Full Mode (scrape + cron + API server)

```bash
# Runs initial scrape, schedules re-scrape every 6 hours, and starts API
npm run dev
```

## REST API Endpoints

Once you run `npm run serve` (port 4100):

| Method | Endpoint | Description |
|---|---|---|
| GET | `/health` | Health check |
| GET | `/api/search?q=solar` | Full-text search |
| GET | `/api/search?q=rwa&category=rwa&region=Europe` | Filtered search |
| GET | `/api/trending?days=7` | Trending articles (last N days) |
| GET | `/api/category/solar` | All articles in a category |
| GET | `/api/region/India` | All articles from a region |
| GET | `/api/report` | Full trend report with stats |
| GET | `/api/stats` | Article counts by category/region/source |
| GET | `/api/jobs` | Recent scrape job history |
| POST | `/api/scrape` | Trigger a scrape `{ "query": "...", "all": true }` |

## Integration with Aethera Server

The pipeline runs independently on port `4100`. The main Aethera server (`server/`) can call it via HTTP:

```ts
// Fetch trending tokenization news for the dashboard
const res = await fetch("http://localhost:4100/api/trending?days=7&limit=10");
const { articles } = await res.json();

// Search for articles relevant to a specific project
const res = await fetch("http://localhost:4100/api/search?q=solar+india&category=solar");
```
